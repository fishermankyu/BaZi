# 调候用神 (Seasonal Adjustment) — The Independent Third Dimension

调候 is a separate lens from both 身强身弱 (`strength_formula.md`) and 従格 (`congge.md`). It asks a different question entirely: not "is the Day Master strong or weak," but "does the chart's season need warming or cooling, and is there something in the chart already doing that job." A chart can be perfectly balanced by 扶抑 standards and still be "cold" or "dry" by seasonal standards, and vice versa — the two dimensions don't substitute for each other.

**Core principle**: 春(spring)=湿(damp), 夏(summer)=暖/燥(warm/dry), 秋(autumn)=燥(dry), 冬(winter)=寒/湿(cold/damp). The month branch sets this backdrop. When the chart lacks anything to counteract an extreme reading — most commonly no fire at all in a winter (亥子丑, or early 寅) chart, or no water at all in a summer (巳午未) chart — 调候 becomes relevant. Source (《穷通宝鉴》): "命中无火，冬生水冷木寒，纵有印比，亦难生发" — even a chart with plenty of 印/比劫 support can fail to "come alive" if it's freezing cold and has nothing to warm it.

Unlike 扶抑 (which just wants *any* generic supporting element), 调候 in the classical 穷通宝鉴 system specifies **which exact heavenly stem** is called for, per Day Master per month — not just "needs fire" but "needs 丙 specifically, with 癸 as secondary." This is why the reference table below is broken out to stem-and-month granularity (十干喜用提要, 120 entries) rather than a general "cold needs fire" rule.

## How to use the table

1. Identify the Day Master and the month branch.
2. Look up the matching Day-Master table below, find the month row.
3. The first-listed stem(s) are the primary 调候 用神; later-listed stems are secondary/supporting, per the classical note in each row.
4. Check whether that stem is actually present (ideally transparent on a heavenly stem) in the chart. If present and unafflicted, 调候 need is already met — don't force additional emphasis on it. If absent, the chart has an unmet 调候 need and this should be flagged.
5. Not every month is equally seasonally urgent — 亥子丑(+early寅) and 巳午未 are where 调候 is most classically emphasized (需要火/需要水的月份); the other months' entries still apply but with somewhat less urgency in most sources.

## Priority: when 调候 and 扶抑/格局 conflict

These are genuinely different dimensions, not a strict hierarchy in the ordinary case: 调候 speaks to something more like the chart's overall "climate/mood," while 扶抑 (from `strength_formula.md`) speaks to the Day Master's raw strength. A chart can need both, and usually satisfying one doesn't automatically satisfy the other.

**But when they do directly conflict** (e.g. the 扶抑 formula wants to drain/control a strong Day Master using an element that would also remove the chart's only source of warmth in a freezing-cold chart), classical sources side with 调候: 《穷通宝鉴》's stance is **"先解冻，后论生克"** — resolve the seasonal extreme first, discuss the generating/controlling relationships (扶抑 logic) second. E.g. even a chart that technically qualifies as a good 财格 (wealth structure) in a freezing winter month still needs 丙 (fire) as the practical first-priority 用神 before the 财格 logic is applied on top.

**Two failure modes to avoid** (both flagged as common mistakes in source material):
1. **Ignoring 调候 entirely** and reasoning purely off 生克/扶抑 — this misses charts where the Day Master is "balanced on paper" but still cold/dry and non-functional.
2. **Over-applying 调候 rigidly** — assuming every winter chart needs fire and every summer chart needs water regardless of what's already present. If the chart already has adequate warming/cooling (e.g. a strong fire pillar already sitting in a winter chart), don't keep insisting on more — the need is already met, and forcing extra weight onto it double-counts a solved problem.

**Practical sequencing for this skill**:
1. Run 扶抑 (`strength_formula.md`) and 従格 (`congge.md`) as already documented.
2. Separately check 调候 using the table above: is the month genuinely seasonally extreme, and is the called-for stem present and functioning?
3. If 调候 need is **unmet** (missing entirely) in a genuinely extreme month (亥子丑巳午未 especially), say so explicitly and note it takes practical priority as "first-priority 用神" per 穷通宝鉴, even layered on top of whatever 扶抑/従格 concluded.
4. If 调候 need is **already met** by something already in the chart, just note that it's satisfied — don't create tension with the 扶抑/従格 reading where none exists.
5. If a genuine conflict exists (扶抑 wants to remove the exact element serving as 调候), flag it explicitly to the user and default to the 调候 priority per §"先解冻，后论生克", while noting this is one classical position (《穷通宝鉴》's), not the only school of thought — some modern practitioners weight this differently case by case.
